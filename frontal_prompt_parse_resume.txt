Two changes needed in the Prescreening workflow, please show me the plan before building either:

1. File parsing for PDF résumés.
The candidate objects from "List Candidates" now include an optional `resume_url` field alongside `profile_text` (real candidates will have `resume_url` pointing at a PDF, since that's how they'll come from the real export; `profile_text` is mostly for hand-typed test candidates).
Add a step in the Loop, right after we get the candidate and before "Format Criteria and Candidate", that:
  a. Branches (Paths/Condition) on whether `candidate.resume_url` is present.
  b. If present: parses `candidate.resume_url` into text using the built-in Parse file action, and feeds that parsed text into "Format Criteria and Candidate" as `cv_text`.
  c. If NOT present: skip parsing, use `candidate.profile_text` as `cv_text`, like it does now.

2. The workflow must decide pass/fail itself, not the dashboard.
I changed the dashboard's endpoint contract on purpose: `POST /api/workflow/candidates/{id}/prescreen-result` now REQUIRES a `passed` boolean in the body, and no longer computes it internally. Reasoning: this dashboard is a stand-in for Ashby, and Ashby has no "judge this score" skill — only "move this application to stage X". So the dashboard should only ever be told what to do, never decide. All decision logic must live in the workflow, the same place it will live once we point this back at real Ashby.
Please add, right after the Scoring Agent and before "Submit Prescreen Result":
  a. A step that reads `pass_threshold` from `GET https://uirecruitingplatform-production.up.railway.app/api/settings` (or wherever "Fetch Evaluation Config" already gets it from — reuse that call if it already includes pass_threshold, don't add a duplicate call).
  b. A Condition/Paths node comparing the Scoring Agent's `score` output to `pass_threshold` (score >= pass_threshold → passed = true, else false).
  c. Update "Submit Prescreen Result" to send `{"score": ..., "rationale": ..., "passed": true|false}` — the boolean coming from this new Condition node, not computed anywhere else.

Show me both plans before building.
